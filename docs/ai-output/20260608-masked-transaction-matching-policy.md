# 마스킹 실거래 매칭 정책

- 작성일: 2026-06-08
- 대상: 마곡동 상업업무용 실거래 대시보드
- 원칙: 디스코, 밸류맵 등 상용 서비스 크롤링은 사용하지 않는다.

## 1단계. 공식 실거래 API

- `PUBLIC_DATA_SERVICE_KEY`로 국토교통부 상업업무용 실거래 API를 월별 재호출한다.
- 정확 지번이 내려오는 거래는 `PARCEL|지번`으로 확정 반영한다.
- `7**`, `8**`, `1***`처럼 마스킹된 지번은 원자료만으로 확정 지번을 복원하지 않는다.

## 2단계. 내부 후보군 유일성 검증

- 건축HUB 전유공용면적 자료를 마곡동 후보 필지 전체로 인덱싱한다.
- 마스킹 거래는 `용도분류 + 층 + 전용면적 + 건축년도 필터`로 후보를 찾는다.
- 후보 필지가 하나일 때만 해당 건물에 `추정`으로 붙인다.
- 같은 조건이 여러 필지에 존재하면 상세에 붙이지 않는다.
- 같은 필지 안에 같은 면적 호실이 여러 개 있으면 건물 추정은 허용하되 호실 확정은 하지 않는다.

## 3단계. 등기기록 확정 대조

- 개별 부동산 등기사항증명서 원문이나 집합건물 호실별 등기기록을 공개 무료 API로 바로 가져오는 일반 경로는 확인되지 않았다.
- 대법원은 등기정보광장처럼 통계/현황성 개방 포털을 운영하지만, 개별 등기부등본 원문 열람/발급은 인터넷등기소 업무에 가깝다.
- 확정 대조가 필요하면 합법적인 열람/발급 파일, 기관 제휴 API, 또는 사용자가 제공한 등기 PDF/이미지를 받아 호실, 접수일, 거래가액을 대조한다.

## 현재 M시그니처 반영 기준

- 정확 지번: 4건.
- 마스킹 추정: 5건.
- 총 상세 반영: 9건.
- 제외된 후보: 8층 일부 후보처럼 같은 조건이 다른 필지에도 존재하는 거래.
